# Agent Instructions

## Version Control

* Use git for version control
* Make commits directly to the current branch for each step in the process
* Keep commits to small, local and focused changes
* Use the Conventional Commit specification
* Maintain a .gitignore file

## Coding Style

* Write idiomatic Go code following standard conventions and patterns
* Add type hints and standard annotations for all new functions
* Add swagger documentation for all API endpoints
* Add comments for complex logic

## Changelog

* Note all changes in the Changelog.md
* Follow the Semantic Versioning specification
* Structure changes by date with newest changes on top

## Testing

* Include unit tests for new functionality
* Ensure test coverage for all exported functions
